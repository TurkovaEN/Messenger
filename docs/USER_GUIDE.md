\# Руководство пользователя



\## Запуск мессенджера



\### 1. Установка ключа шифрования



Перед запуском сервера и всех клиентов необходимо установить переменную окружения MSG\_KEY.



Linux:

export MSG\_KEY="my\_super\_secret\_key\_32\_bytes\_len!!"



Windows (cmd):

set MSG\_KEY=my\_super\_secret\_key\_32\_bytes\_len!!



Внимание: Ключ должен быть одинаковым на сервере и всех клиентах!



\### 2. Запуск сервера



./server/server 5555 server/logs/server.log



Параметры:

&#x20; 5555 - порт для подключений

&#x20; server/logs/server.log - файл лога



Сервер работает до нажатия Ctrl+C.



\### 3. Запуск консольного клиента



./client/client <хост> <порт> <имя\_пользователя>



Пример:

./client/client 127.0.0.1 5555 Alice



\### 4. Запуск графического клиента (Qt)



Запустите исполняемый файл:

./client\_qt/build/messenger\_qt\_client



При первом запуске появится окно входа:

&#x20; - Введите имя пользователя

&#x20; - Введите адрес сервера (например, 127.0.0.1:5555)



\## Команды



/create <имя>   - создать комнату

/join <имя>     - войти в комнату

/leave          - покинуть комнату

/rooms          - список всех комнат на сервере

/help           - показать справку

/quit или /exit - выйти из клиента



\## Пример сессии



Пользователь Alice:

/create java\_chat

Создана комната: java\_chat



/join java\_chat

Вы вошли в комнату: java\_chat



Всем привет!

\[Alice]: Всем привет!

/join java\_chat

Вы вошли в комнату: java\_chat

Привет!

\## Шифрование



Все сообщения шифруются AES-256-CBC перед отправкой. При перехвате трафика сообщения выглядят как base64-строки.



Пример зашифрованного сообщения:

type=enc;data=U2FsdGVkX1/3j5k7G8pQrZyLmNvBw4tY6uIxO...



\## Решение проблем



Ошибка "Cannot open users file"



Решение: создать папки вручную

mkdir -p server/data server/logs

touch server/data/users.txt



Ошибка "bind failed, err=10013" (Windows)



Решение: порт занят, используйте другой порт

server\_win.exe 5556 server/logs/server.log



Qt-клиент не запускается (ошибка DLL)



Решение: запустить windeployqt в командной строке Qt

windeployqt messenger\_qt\_client.exe



Не подключается к серверу



1\. Проверьте, запущен ли сервер

2\. Проверьте правильность IP и порта

3\. Проверьте брандмауэр (порт 5555 должен быть открыт)
